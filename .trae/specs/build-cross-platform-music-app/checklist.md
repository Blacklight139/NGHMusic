# Checklist

## 架构与后端
- [x] 六端均可运行，各端调用各自原生音频播放组件（桌面系统组件/AVPlayer/ExoPlayer/HarmonyOS AVPlayer）
- [x] 业务逻辑、音源引擎、协议客户端、缓存层均在共享 Rust 核心实现，各端经 FFI 调用
- [x] 同一音源在六端导入后，搜索/元数据/缓存行为一致

## 音源系统
- [x] 标准 JSON Schema（`schemas/sound-source.schema.json`）已定义并被核心校验器使用
- [x] 设置内可导入 JSON 音源，合法音源校验通过并启用
- [x] 非法/不可适配音源提示具体错误且不加载
- [x] 社区音源经适配层迁移后可加载，并提示迁移结果
- [x] 音源支持启用/禁用/切换/优先级
- [x] 通过音源 API 可获取结构化元数据（标题/艺术家/专辑/封面/时长/歌词地址）
- [x] 搜索点击后定位音源播放数据并缓存播放
- [x] 缓存命中时优先本地播放

## 核心功能
- [x] 播放组件提供：播放/暂停/上一首/下一首/进度拖动/音量/播放模式
- [x] 播放状态重启后持久化
- [x] 播放列表支持添加/移除/清空/拖动排序
- [x] 收藏夹支持多分组/添加/移除/导入/导出
- [x] 歌词滚动支持解析/同步高亮/点击跳转/可选翻译
- [x] 排行榜可展示并点击播放
- [x] 搜索跨音源聚合，支持歌曲/专辑/艺术家分类与分页，结果含音源来源

## 本地音乐播放
- [x] 设置内可添加/移除本地音乐目录，并触发递归扫描
- [x] 扫描支持 mp3/flac/m4a/ape/ogg/wav/aac 等格式，展示扫描进度与结果
- [x] 解析音频内嵌标签（ID3/FLAC/MP4/APE）得到标题/艺术家/专辑/封面/时长，缺失字段以文件名回退
- [x] 本地音乐库持久化索引（SQLite）
- [x] 文件夹监听做增量同步（新增/删除/移动），不全量重扫
- [x] 本地音乐作为内置源接入聚合搜索/播放列表/收藏夹，结果标注「本地」来源
- [x] 本地音乐库可按歌曲/专辑/艺术家/文件夹浏览并播放（四端）
- [x] 本地音乐通过各端原生音频组件直接读取本地文件播放，无需联网与缓存

## UI/UX
- [x] 简约风格设计系统落地（色彩/字体/间距/图标）
- [x] 四端 UI 视觉协调统一
- [x] 页面切换平滑稳定，无闪烁/卡顿
- [x] 响应式适配（桌面窗口/移动竖屏）

## NAS 与协议
- [x] 飞牛 API 集成：可配置 NAS 并播放歌曲
- [x] 飞牛 API 失败时提示并支持重试，不崩溃
- [x] SMB 协议：可添加源并浏览/播放（注：协议客户端为占位，需启用 pavao feature + libsmbclient 系统库后接通真实传输）
- [x] WebDAV 协议：可添加源并浏览/播放
- [x] FTP 协议：可添加源并浏览/播放
- [x] DLNA 协议：可添加源并浏览/播放（注：协议客户端为占位，需集成 DLNA 栈（SSDP/SOAP）后接通真实传输）
- [x] NFS 协议：可添加源并浏览/播放（注：协议客户端为占位，需 NFS 客户端库后接通真实传输）
- [x] 缓存管理 UI 可查看容量并清理
