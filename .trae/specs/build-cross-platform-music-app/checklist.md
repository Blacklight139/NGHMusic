# Checklist

## 架构与后端
- [ ] 六端均可运行，各端调用各自原生音频播放组件（桌面系统组件/AVPlayer/ExoPlayer/HarmonyOS AVPlayer）
- [ ] 业务逻辑、音源引擎、协议客户端、缓存层均在共享 Rust 核心实现，各端经 FFI 调用
- [ ] 同一音源在六端导入后，搜索/元数据/缓存行为一致

## 音源系统
- [ ] 标准 JSON Schema（`schemas/sound-source.schema.json`）已定义并被核心校验器使用
- [ ] 设置内可导入 JSON 音源，合法音源校验通过并启用
- [ ] 非法/不可适配音源提示具体错误且不加载
- [ ] 社区音源经适配层迁移后可加载，并提示迁移结果
- [ ] 音源支持启用/禁用/切换/优先级
- [ ] 通过音源 API 可获取结构化元数据（标题/艺术家/专辑/封面/时长/歌词地址）
- [ ] 搜索点击后定位音源播放数据并缓存播放
- [ ] 缓存命中时优先本地播放

## 核心功能
- [ ] 播放组件提供：播放/暂停/上一首/下一首/进度拖动/音量/播放模式
- [ ] 播放状态重启后持久化
- [ ] 播放列表支持添加/移除/清空/拖动排序
- [ ] 收藏夹支持多分组/添加/移除/导入/导出
- [ ] 歌词滚动支持解析/同步高亮/点击跳转/可选翻译
- [ ] 排行榜可展示并点击播放
- [ ] 搜索跨音源聚合，支持歌曲/专辑/艺术家分类与分页，结果含音源来源

## UI/UX
- [ ] 简约风格设计系统落地（色彩/字体/间距/图标）
- [ ] 四端 UI 视觉协调统一
- [ ] 页面切换平滑稳定，无闪烁/卡顿
- [ ] 响应式适配（桌面窗口/移动竖屏）

## NAS 与协议
- [ ] 飞牛 API 集成：可配置 NAS 并播放歌曲
- [ ] 飞牛 API 失败时提示并支持重试，不崩溃
- [ ] SMB 协议：可添加源并浏览/播放
- [ ] WebDAV 协议：可添加源并浏览/播放
- [ ] FTP 协议：可添加源并浏览/播放
- [ ] DLNA 协议：可添加源并浏览/播放
- [ ] NFS 协议：可添加源并浏览/播放
- [ ] 缓存管理 UI 可查看容量并清理

## 文档
- [ ] 音源开发文档含 Schema 说明、元数据/搜索/缓存对接、社区迁移方案
- [ ] API 文档为 Markdown、仿 Apifox 规格，覆盖音源/飞牛/协议接口
- [ ] API 文档含请求/响应示例与错误码

## 代码质量与 Bug 排查
- [ ] 静态分析工具链配置（Rust/桌面/iOS/Android/HarmonyOS）
- [ ] 边界条件与错误处理审查通过
- [ ] 内存泄漏排查通过（Rust 生命周期/各端资源释放）
- [ ] 状态管理与异步竞态排查通过
- [ ] 兼容性与性能瓶颈测试通过（六端真机/模拟）
- [ ] Bug 报告含复现步骤/预期/实际/优先级并已分类
